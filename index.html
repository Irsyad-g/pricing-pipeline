Architecture

Pipeline overview

Seven stages from raw input to published prices — each stage is a separate Python module.

01
Raw data ingestion
Reads daily usage files (DAILY_USAGE_*.xlsx, BSN-*.xlsx) and subscription records. Normalises ICCID format, parses MCC-based area codes, converts usage to MB.
02
Subscription processor
Joins each subscription window to its daily usage records. Computes total usage, usage ratio, actual active days, and country-level consumption distribution per ICCID.
03
Behaviour factor engine
Aggregates usage ratios at package level using P50/P75 blend. Builds lookup tables for 5-tier interpolation. Applies QFF clamping and isotonic smoothing.
04
Full SKU matrix generator
Generates all combinations of (country × quota × duration days 1–30) for every known country. Countries with no historical data can be force-included via config.
05
Score interpolation
Assigns a behaviour score and confidence level to every SKU in the matrix using the 5-tier fallback chain. Stores source tier and sample count for downstream weighting.
06
Pricing calculation
Computes weighted modal cost per package from MCC-resolved country rates. Applies product-type-specific pricing formula. Outputs SIM, eSIM, and flat prices in IDR.
07
Google Sheets export
Clears and repopulates the PRICING_OUTPUT worksheet with the full SKU price table. Also exports summary reports and behaviour data to Excel files.

Key components

How it works

Behaviour score interpolation

Every SKU in the 10,800+ matrix gets a behaviour score via a 5-tier fallback chain. The system tries the most specific source first, falling back to increasingly general estimates.

Tier Source Condition QFF weight
EXACT best Same country + duration + quota ≥ 15 samples 8–35% QFF
REGION good Same region, same duration + quota Any 40% QFF
GLOBAL ok All countries, same duration + quota Any 55% QFF
NEAREST ok Closest duration (±5 days), same quota Any 70% QFF
CURVE fallback Mathematical formula Always 90% QFF
⚖

Quota Feasibility Factor

Physics-informed clamp that prevents unrealistic scores. BIG DATA uses calibrated daily consumption reference. FUP and Unlimited use duration-decay curves derived from actual activity rates.

🌍

Weighted modal cost

Dynamic supplier cost per package — not a flat table. MCC codes from daily data resolve to countries, each with a CNY/GB rate. Cost is usage-weighted across all roaming destinations.

📐

Market-calibrated formula

BIG DATA pricing reverse-engineered from competitor data. Log-linear model fit with 5.7% mean error: rate = 0.5848 + 0.065·ln(day) − 0.1016·ln(GB). Reduced overpricing from +53% to target 10–15%.

〰

Isotonic smoothing

BIG DATA scores are monotonically non-decreasing by duration. FUP and Unlimited are monotonically decreasing — engagement drops over longer subscriptions. Enforced via scikit-learn IsotonicRegression.

Pricing formula by product type

# BIG DATA — market-calibrated log-linear
rate = 0.5848 + 0.065 * np.log(day) - 0.1016 * np.log(gb)
modal_adj = modal * gb * behaviour_score * rate

# FUP / Unlimited — behaviour score with duration decay
modal_adj = modal * behaviour_score * days * quota_gb

# Final price (IDR)
cost_idr = (modal_adj + device_cost) * cny_rate
price = margin(cost_idr)  # rounds to nearest 10,000 + 9,000
